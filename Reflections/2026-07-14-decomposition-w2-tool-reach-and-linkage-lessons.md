## Patterns and Root Causes

The night's unifying pattern is [REFL-011]'s tool-reach class — a verification tool's green is a state claim bounded by the tool's REACH — and it fired four distinct times in one session: (1) SwiftPM's native unused-dep warning ([PKG-DEP-007]'s "authoritative signal") reaches in-package product references only; it is structurally blind to downstream transitive member-access linkage, and the pruned "unused" dep was load-bearing for an ungreppable consumer's member lookups. (2) A resolve-then-build oracle reaches only the existing pin-set: `swift package resolve` never advances branch pins, so the first oracle green verified the pre-thin donor. (3) A whole-file closure grep reaches prose as well as dependency spellings — two comment-line "coenttb" hits were noise the guard should never have counted. (4) The evidence build correctly reached tf's own scope, which is precisely why its verdict needed an explicit exception note for scopes it cannot reach. The deeper mechanism under (1): pre-MemberImportVisibility member lookup makes a target's LINKED module set part of its de-facto public surface — [HANDOFF-040]'s "member accesses don't grep" extends past greppable consumers to pure linkage, and the only detector for an ungreppable consumer is an honest oracle build.

Second pattern: plan rows falsify on source contact at a steady rate — W1 had three falsified rows, W2 had three more (consumer count, parser shape, Sendable transfer), and R-5's entire premise inverted. Inventory-derived plans decay in days in this ecosystem's current churn; the charter clause that makes verification a gate rather than a virtue is what converts these from mid-wave breakage into pre-edit corrections. Third pattern: subagent lane mortality has a consistent signature — death comes silently AFTER authoring, before gating (work visible on disk, no commit, no processes) — and the coordinator-side mitigation (per-event notification + status ping + bounded fallback timer + verified adoption) worked exactly as the gotcha table prescribes. Fourth, smaller: premise inversions travel upward fast when the evidence artifact is mandatory — the R-5 table let the principal re-derive the architecture (URI/URL as deliberate pair, WHATWG L2 gap) in one round instead of re-investigating.
