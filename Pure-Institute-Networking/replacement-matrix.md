# External-to-Institute replacement matrix

## Reading the matrix

“Active” means reachable from the selected `com_repotraffic_app` production product unless a row explicitly says repo-wide or marketing executable. An absent import does not make an active product stale. The 48 Vapor imports split into 32 app-path and 16 marketing-executable imports; both executables must migrate (or the second must be removed) so their manifest edges cannot retain the old family. Acceptance means zero forbidden identity in the release products, manifest/lock, container, and SBOM. Differential fixtures belong to a separate comparison root and lock. “Gate” is the last condition before removing the named product/package edge; shared final gates also require the exact release closure proof in the validation program.

## Shared final closure gate

For every row marked removed:

```sh
swift package show-dependencies --format json
swift package dump-package
Scripts/selected-product-closure --package-path . --lock Package.resolved --product com_repotraffic_app --output Artifacts/closure/com_repotraffic_app.json
Scripts/selected-product-closure --package-path . --lock Package.resolved --product com_repotraffic --output Artifacts/closure/com_repotraffic.json
swift build --product com_repotraffic_app -c release -v
swift build --product com_repotraffic -c release -v
rg -n -i '(vapor|boiler|server-foundation|postgres-nio|swift-nio|nio-ssl|nio-http2|nio-extras|nio-transport-services|async-http-client|websocket-kit|routing-kit|multipart-kit|console-kit|async-kit|queues|redistack)' Package.swift Package.resolved
rg -n --pcre2 --glob '*.swift' '^[\t ]*(?:@[_A-Za-z][_A-Za-z0-9]*(?:\([^)]*\))?[\t ]+)*(?:(?:private|fileprivate|internal|package|public|open)[\t ]+)?import[\t ]+(?:(?:class|enum|func|protocol|struct|typealias|var|let)[\t ]+)?(?:Vapor|PostgresNIO|_?NIO[_A-Za-z0-9]*|AsyncHTTPClient|Queues|QueuesRedisDriver|RediStack|Boiler|HTML_Vapor|ServerFoundation(?:EnvVars|Vapor)?)\b' Sources Tests
```

`Scripts/selected-product-closure` is a required Wave 0 deliverable, not an existing command silently assumed by this plan. Its versioned canonical JSON records schema/tool version, root package/product, toolchain/platform/traits, release-lock SHA-256, every package identity and locked revision, selected product/target edges, edge condition, and a reachability reason. It recursively reads the root and locked-checkout manifests and fails closed on unresolved products, unsupported conditions, duplicate identities, or lock mismatch. Production, conditional, trait-gated, and test-only fixture graphs must prove inclusion/exclusion behavior.

`show-dependencies` remains only the resolved-package superset; `dump-package` remains only the local manifest. The exact verbose product build/link inputs, binary symbols, and container/SBOM backstop the versioned traversal. A zero from any one search is insufficient; known import fixtures covering attributes (`@preconcurrency`, `@_exported`), access levels (`public`, `internal`, `package`), and scoped imports prove the source scanner is functioning. Comparison roots and locks are archived separately and never supplied to release-closure commands.
